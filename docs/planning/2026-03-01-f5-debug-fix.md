# F5 デバッグ修正 + テスト自動化

## Context

claude-resurrect 拡張の F5 デバッグで Extension Development Host にワークスペースフォルダが開かない問題。
手探りデバッグのループに陥り、根本原因の特定なしに試行錯誤を繰り返した反省から、
原因調査 → 計画 → レビュー → 実装の順序を徹底する。

## 根本原因

**launch.json の `args` でフォルダパスが `--extensionDevelopmentPath` の後ろにあった。**

VSCode CLI は最初の非フラグ引数をフォルダとして解釈する。フラグの後ろに置くと無視される。

確認元: [microsoft/vscode/.vscode/launch.json](https://github.com/microsoft/vscode/blob/main/.vscode/launch.json)

```json
// VSCode 自身のテスト構成（正しい順序）
"args": [
    "${workspaceFolder}/extensions/vscode-api-tests/testWorkspace",  // フォルダが先
    "--extensionDevelopmentPath=${workspaceFolder}/extensions/vscode-api-tests"
]
```

launch.json 修正が効くか確認。F5 → Extension Development Host でフォルダが開き、

### 2. テスト自動化（再発防止）

#### 批評レビューの指摘

> `@vscode/test-electron` + Mocha は Vitest との2ランナー共存になりコスト過剰。
> Vitest + vscode モックの方が軽量。
> launch.json の再発防止はコメント1行で十分かもしれない。

#### 判断

批評の指摘を受け入れる。`@vscode/test-electron` は導入しない。

代わりに:
- **launch.json にコメント追加**: args 順序の根拠を明記
- **Vitest + vscode モック**: `activate()` のユニットテストを追加。コマンド登録・ステータスバー生成を検証

#### 対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `.vscode/launch.json` | コメント追加（args 順序の根拠） |
| `src/extension.test.ts` | 新規: activate() のユニットテスト（vscode モック） |

### 3. コンパイル + テスト

```bash
npm run typecheck && npm run lint && npm run test && npm run compile
```
